# FP 산정 상세 레퍼런스

## 1. 기능유형 식별 기준

### ILF (내부논리파일) 식별
- 애플리케이션 경계 내에서 **유지(CUD)** 되는 데이터 그룹
- 예: 회원정보, 주문, 게시판, 설정, 이력/로그, 권한, 코드관리
- **식별 팁**: ERD의 주요 엔티티, 관리자 화면에서 CRUD하는 데이터

### EIF (외부연계파일) 식별
- 외부 시스템이 유지하며 우리 시스템은 **참조만** 하는 데이터
- 예: 행정표준코드, 우편번호DB, 외부 인증정보, 타 시스템 마스터
- **식별 팁**: 연계 인터페이스 목록, 외부 API로 조회하는 데이터

### EI (외부입력) 식별
- ILF를 생성/수정/삭제하는 기능 (각각 별개 EI)
- 예: 등록, 수정, 삭제, 파일 업로드, 승인/반려, 상태변경
- **식별 팁**: 화면의 저장/삭제/처리 버튼 → 각각 1 EI

### EO (외부출력) 식별
- 데이터를 **가공/계산**하여 출력하는 기능
- 예: 통계 보고서, 대시보드 차트, 합계/평균, 엑셀 다운로드, PDF 생성, 이메일 발송
- **식별 팁**: 집계/계산/변환 로직이 있으면 EO

### EQ (외부조회) 식별
- 데이터를 가공 없이 **그대로 조회**하는 기능
- 예: 목록 조회, 상세보기, 검색(필터), 코드 조회, 팝업 조회
- **식별 팁**: SELECT 후 그대로 표시하면 EQ

## 5. Sanity Check 기준

| 검증 항목 | 정상 범위 | 이상 판단 |
|---------|---------|---------|
| 화면당 평균 FP | 5~15 FP | 3 미만 또는 20 초과 시 재검토 |
| ILF : EI 비율 | 1 : 2~4 | 비율 벗어나면 재검토 |
| EQ 비율 | 전체의 30~50% | 극단값이면 재검토 |
| FP당 실효 단가 | 40~60만 원 | 30만 원 미만이면 과소 산정 의심 |

## 6. 업무 모듈별 평균 FP 참고치

아래는 유사 공공사업에서 관측되는 업무 모듈별 평균 FP 참고치이다.
실제 산정 시에는 반드시 요구사항별로 개별 식별해야 한다.

| 업무 모듈 | 평균 FP 범위 | 구성 예시 |
|---------|------------|---------|
| 사용자 관리 | 25~40 FP | 회원가입, 로그인, 프로필, 권한 |
| 게시판/커뮤니티 | 20~35 FP | 목록, 상세, 작성, 수정, 삭제, 댓글, 첨부 |
| 통계/대시보드 | 30~60 FP | 현황판, 차트, 보고서, 엑셀 다운로드 |
| 결재/워크플로우 | 40~70 FP | 기안, 결재, 반려, 위임, 이력 |
| 연계/인터페이스 | 15~30 FP/건 | API 호출, 데이터 동기화, 파일 연계 |
| 관리자 기능 | 20~40 FP | 코드관리, 메뉴관리, 로그, 시스템설정 |
| 검색/조회 | 10~25 FP | 통합검색, 상세검색, 필터 |
