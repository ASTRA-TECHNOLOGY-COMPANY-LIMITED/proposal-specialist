# 섹션 유형별 글쓰기 전략

제안서 섹션을 작성할 때, 섹션 제목을 아래 유형 중 하나로 분류하고 해당 전략을 적용하라.
상세 전략은 이 디렉토리의 `type-strategies.md`를 참조하라.

## 유형 분류 빠른 참조

- **A**: 사업이해도 / 과업이해 / 현황분석 — AS-IS → 문제 → TO-BE → 제안 방향
- **B**: 추진전략 / 기대효과 / 사업방향 — 차별화 3축 헤드라인 + 근거
- **C**: 적용기술 / 기술방안 / 기술성 — 요구사항 1:1 대응표 + 아키텍처
- **D**: 표준프레임워크 / 개발방법론 — 표준 준수 + 방법론 + 스프린트
- **E**: 수행조직 / 추진체계 / 조직도 — 조직 시각화 + 인력 프로필 + R&R
- **F**: 일정계획 / WBS — 마일스톤 + 산출물 + 보고 일정
- **G**: 품질관리 / 보안관리 / 리스크관리 — 체계 → 식별 → 평가 → 대응
- **H**: 유사실적 / 수행실적 — 3건 이상 상세 + 정량 성과
- **I**: 기대효과 / 발전방안 — 정량+정성 분리, 발주기관 목표 연결
- **J**: 교육훈련 / 기술이전 — 교육 계획표 + 기술이전 체크리스트
- **K**: 하자보수 / 유지보수 — SLA 수준표 + 대응 절차

## 적용 방법

1. 섹션 제목과 목차 메타데이터를 기반으로 위 유형 A~K 중 하나를 선택한다
2. `type-strategies.md`에서 해당 유형의 **구조**, **필수 도표**, **필수 도안**, **주의사항**을 확인한다
3. 유형별 구조를 따라 섹션 콘텐츠를 작성한다
