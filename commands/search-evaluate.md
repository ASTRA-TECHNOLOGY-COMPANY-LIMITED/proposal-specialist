# 제안 적합성 평가

$ARGUMENTS 에 지정된 공고에 대해 상세 분석하고 제안 전략을 수립하라.

## 인자 파싱 및 공고 확보

다음 순서로 분석 대상 공고를 확보한다:

1. **$ARGUMENTS에서 확인**: 입찰공고번호, 공고 URL, 또는 RFP 파일 경로가 있으면 사용한다
2. **이전 결과 활용**: 인자가 없으면 현재 대화에서 이전 `search-bid` 결과가 있는지 확인하고, 있으면 목록을 보여주며 사용자에게 선택을 요청한다
3. **사용자에게 요청**: 위 방법으로도 대상을 확보하지 못하면 대화형으로 요청한다:
   "분석할 공고 정보를 입력해주세요. 다음 중 하나를 입력할 수 있습니다:
   - 입찰공고번호 (예: 20260101001)
   - RFP/제안요청서 파일 경로 (예: data/business/사업명/제안요청서.pdf)
   - 공고 URL"
   - 사용자가 입력하면 해당 정보로 계속 진행한다

## 평가 절차

1. MCP Tool로 공고 상세 정보를 조회한다 (g2b_get_bid_detail 등)
2. **입찰 마감일을 확인한다**
   - `bidClseDt` 필드를 확인하여 마감 여부를 판단한다
   - 이미 마감된 공고인 경우 사용자에게 즉시 경고한다:
     "이 공고는 {마감일}에 마감되었습니다. 그래도 분석을 진행하시겠습니까?"
   - 사용자가 확인 후에만 분석을 진행한다 (참고용 분석)
   - `deadlineStatus`가 `urgent` (3일 이내)인 경우에도 경고:
     "마감까지 {N}일 남았습니다. 준비 기간이 부족할 수 있습니다."
3. 첨부파일 URL을 확인하고 다운로드한다 (download_attachment Tool)
4. 다운로드된 파일 경로를 확인한다 (list_downloads Tool)
5. **Read Tool로 다운로드된 각 파일의 내용을 읽는다**
   - PDF: Read Tool (pages 파라미터로 분할 읽기, 20페이지 단위. 100페이지 초과 시 목차/요구사항/평가기준 섹션 선택 읽기)
   - 텍스트/CSV: Read Tool (직접 읽기)
   - HWP/DOCX: Read Tool 시도. 읽기 불가 시 사용자에게 "HWP 파일은 직접 분석이 어렵습니다. PDF로 변환 후 다시 시도해주세요"를 안내
   - 첨부파일이 없는 경우: "첨부파일이 없습니다. 공고 상세 정보만으로 분석합니다"를 안내
6. rfp-evaluator 에이전트를 호출하여 상세 분석을 위임한다
   - 공고 상세 정보 + 첨부파일 내용을 함께 전달
7. 평가 보고서를 작성한다

## 평가 항목

| 항목 | 배점 | 평가 기준 |
|---|---|---|
| 자격 요건 충족 | 20점 | 참가 자격, 면허, 인증 |
| 기술 역량 매칭 | 30점 | 요구 기술과 보유 기술 일치도 |
| 유사 실적 | 25점 | 관련 프로젝트 경험 |
| 가격 경쟁력 | 15점 | 추정가격 대비 적정성 |
| 일정 적합성 | 10점 | 준비 기간, 수행 기간 |
