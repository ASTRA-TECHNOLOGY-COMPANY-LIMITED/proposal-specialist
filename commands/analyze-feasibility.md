# 사업성 종합 검토

$ARGUMENTS 에 지정된 문서(RFP, 입찰공고, 사업계획서)를 대상으로
**Shipley Capture Planning 프레임워크** 기반의 종합 사업성 검토를 수행하라.

## 인자 파싱 및 문서 확보

$ARGUMENTS 형식: `<문서경로 또는 입찰공고번호> [시드경로]`

다음 순서로 분석 대상 문서를 확보한다:

1. **$ARGUMENTS에서 확인**: 파일 경로, 입찰공고번호가 있으면 사용한다. 두 번째 인자가 있으면 기업 시드 경로로 사용한다.
2. **자동 탐색**: 인자가 없으면 다음을 순차 탐색한다
   - 현재 대화에서 이전 `search-evaluate` 또는 `analyze-risk` 결과가 있으면 해당 공고 정보를 활용한다
   - `data/business/` 디렉토리에서 Glob Tool로 RFP/제안요청서 파일 검색
   - `data/output/` 디렉토리에서 최근 생성된 목차.md 또는 fp-analysis.md 검색
3. **사용자에게 요청**: 위 방법으로도 문서를 찾지 못하면 대화형으로 요청한다:
   "사업성 검토 대상 문서를 찾지 못했습니다. 다음 중 하나를 입력해주세요:
   - RFP/제안요청서 파일 경로 (예: data/business/사업명/제안요청서.pdf)
   - 사업계획서 파일 경로
   - 입찰공고번호 (예: 20260101001)"
   - 사용자가 입력하면 해당 정보로 계속 진행한다

**기업 시드 문서 확보** (선택):
- 두 번째 인자로 시드 경로가 제공되면 사용한다
- 없으면 `data/seed/` 디렉토리에서 Glob Tool로 가용 시드를 탐색한다
- 시드가 여러 개 발견되면 목록을 보여주고 사용자에게 선택을 요청한다
- 시드를 찾지 못하면 사용자에게 요청한다:
  "기업 시드 문서가 있으면 경로를 입력해주세요. 없으면 Enter를 눌러 건너뛰세요. (시드 없이도 분석 가능)"
- 시드가 없어도 분석은 진행 가능하다 (역량 매칭 항목은 "확인 필요"로 표시)

## 분석 절차

### Step 1: 문서 수집 및 읽기

- 파일 경로가 확보된 경우: Read Tool로 내용을 읽는다
  - PDF: pages 파라미터로 20페이지 단위 분할 읽기
  - 100페이지 초과 PDF: 목차, 요구사항, 평가기준, 계약조건, 예산 섹션 선택 읽기
  - HWP/DOCX: Read Tool 시도. 읽기 불가 시 PDF 변환 안내
- 입찰공고번호가 확보된 경우: g2b_get_bid_detail MCP Tool로 상세 정보 조회
  - 입찰 마감일 확인: 마감된 공고는 즉시 경고
  - 첨부파일 확인 후 download_attachment로 다운로드
### Step 2: 사업 기본 정보 정리

문서에서 아래 정보를 추출하여 사용자에게 확인받는다:

```
- 사업명:
- 발주기관:
- 추정가격/사업비:
- 수행기간:
- 입찰방식: (협상/적격심사/제한경쟁/2단계)
- 평가방식: (기술:가격 비율)
- 입찰마감일:
```

### Step 3: 분석 범위 확인

사용자에게 분석 범위를 확인한다:

- **Full Review** (기본): 5대 분석 영역 전체
  1. 자격 요건 스크리닝
  2. 기술 역량 매칭
  3. 수익성 시뮬레이션
  4. 리스크 평가
  5. 경쟁 환경 분석

- **Quick Review**: 자격 요건 + Bid/No-Bid 점수만 산출
- **선택 분석**: 사용자가 지정한 영역만

### Step 3.1: 기업 시드 문서 로딩

Full Review 또는 선택 분석(기술 역량, 경쟁 환경 포함) 시에만 기업 시드 문서를 로딩한다.
Quick Review에서는 시드 로딩을 건너뛴다.

- 인자 파싱 단계에서 시드가 이미 확보된 경우: Read Tool로 기업 역량 정보를 읽는다
- 시드가 아직 확보되지 않은 경우: `data/seed/` 디렉토리에서 Glob Tool로 재탐색한다
  - 시드가 발견되면 목록을 보여주고 사용자에게 선택을 요청한다
  - 시드를 찾지 못하면 사용자에게 요청한다:
    "기술 역량 매칭을 위해 기업 시드 문서가 필요합니다. 경로를 입력하거나, Enter를 눌러 건너뛰세요."
- 시드가 없어도 분석은 진행 가능 (역량 매칭 항목은 "확인 필요"로 표시)

### Step 4: feasibility-reviewer 에이전트 호출

수집된 문서 내용, 기업 시드 데이터, 분석 범위를 feasibility-reviewer 에이전트에 위임한다.

에이전트는 다음 6개 Phase를 수행한다:
- Phase 1: 사업 맥락 파악 및 No-Go 트리거 점검
- Phase 2: 5대 영역 상세 분석
- Phase 3: PWin(승률) 산출
- Phase 4: 수익성 시뮬레이션 (Best/Expected/Worst)
- Phase 5: 기대가치(EV) 계산
- Phase 6: Bid/No-Bid 최종 권고

## 후속 커맨드 연계

분석 결과에 따라 사용자에게 후속 작업을 안내한다:

- **적극 참여/조건부 참여**: "제안서 작성을 시작하시겠습니까?" → `/proposal-specialist:generate-toc`
- **FP 기반 상세 원가 분석**: → `/proposal-specialist:analyze-fp`
- **리스크 상세 분석**: → `/proposal-specialist:analyze-risk`
- **불참 권고**: 불참 사유 요약 제공, Decline Letter 초안 작성 제안
