# 입찰 참여 리스크 분석

$ARGUMENTS 에 지정된 문서(RFP, 사업계획서, 제안서, 공고 상세정보)를 분석하여
이 사업에 참여하는 기업이 직면하는 리스크를 체계적으로 식별, 평가, 대응 방안을 수립하라.

## 입력 유형

다음 중 하나 이상의 입력을 받는다:
- RFP/제안요청서 파일 경로 (PDF, HWP, DOCX)
- 사업계획서 파일 경로
- 입찰공고번호 (G2B MCP Tool로 상세 조회)
- 이전 evaluate 결과의 공고 정보
- 기업 시드 문서 경로 (선택, 기업 역량 대비 리스크 매칭에 사용)

## 분석 절차

### 1. 문서 수집 및 읽기
- 파일 경로가 제공된 경우: Read Tool로 내용을 읽는다
  - PDF: pages 파라미터로 20페이지 단위 분할 읽기
  - 100페이지 초과 PDF: 목차, 요구사항, 평가기준, 계약조건 섹션 선택 읽기
  - HWP/DOCX: Read Tool 시도. 읽기 불가 시 PDF 변환 안내
- 입찰공고번호가 제공된 경우: g2b_get_bid_detail MCP Tool로 상세 정보 조회
- 기업 시드 문서가 제공된 경우: Read Tool로 기업 역량 정보 확보

### 2. 리스크 식별 범위 확인
사용자에게 분석 범위를 확인한다:
- **전체 분석** (기본): 6대 리스크 카테고리 전체
- **선택 분석**: 사용자가 지정한 카테고리만

### 3. risk-analyzer 에이전트 호출
수집된 문서 내용과 분석 범위를 risk-analyzer 에이전트에 위임한다.
에이전트는 다음을 수행한다:
- 6대 리스크 카테고리별 리스크 식별
- 5x5 확률-영향 매트릭스 기반 스코어링
- Bid/No-Bid 의사결정 지원 점수 산출
- 카테고리별 완화 전략 수립

### 4. 결과 보고서 작성
에이전트의 분석 결과를 종합하여 아래 형식으로 보고서를 작성한다.

## 6대 리스크 카테고리

| 카테고리 | 영문 | 분석 대상 |
|----------|------|----------|
| 자격/탈락 리스크 | Qualification Risk | 참가 자격, 인증, 실적 요건 미충족 |
| 재무 리스크 | Financial Risk | 보증금, 지체상금, 현금흐름, 원가 초과 |
| 기술 리스크 | Technical Risk | 미검증 기술, 통합 복잡도, 보안 인증 |
| 이행 리스크 | Performance Risk | 납기, 인력, 범위 확대, 하도급 관리 |
| 법적/규제 리스크 | Legal/Regulatory Risk | 부정당업자 제재, 하도급법, 개인정보보호 |
| 전략적 리스크 | Strategic Risk | 기업 전략 정합성, 경쟁 환경, 기회비용 |
