# 제안 전략 수립 (전체 워크플로우)

기업 문서를 기반으로 공고 탐색부터 제안 전략 수립까지 전체 과정을 수행하라.

## Phase 1: 기업 분석

### Step 1: 기업 문서 확보 및 분석

다음 순서로 기업 문서 경로를 확보한다:

1. **$ARGUMENTS에서 확인**: 인자가 있으면 해당 경로를 사용한다
2. **자동 탐색**: 인자가 없으면 다음을 순차 탐색한다
   - `data/seed/` 디렉토리에서 Glob Tool로 기업 시드 문서 검색
   - 환경 변수 `COMPANY_DOCS_DIR`에 지정된 디렉토리
   - 현재 디렉토리에서 기업 소개서나 기술 문서 검색
3. **사용자에게 요청**: 위 방법으로도 문서를 찾지 못하면 대화형으로 요청한다:
   "기업 문서를 찾지 못했습니다. 기업 소개서 또는 기술 문서의 경로를 입력해주세요. (파일 또는 디렉토리 경로)"
   - 사용자가 경로를 입력하면 해당 경로로 계속 진행한다

확보한 기업 문서를 doc-analyzer 에이전트로 분석한다:
- 기업 프로필 생성
- 검색 키워드 도출 (10~15개)
- 적합 사업 유형 결정

### Step 2: 사용자 확인
도출된 키워드와 기업 프로필을 보여주고 수정/보완 의견을 받아라.
"다음 키워드로 검색하겠습니다. 수정하거나 추가할 키워드가 있나요?"

## Phase 2: 공고 탐색

### Step 3: 공고 탐색
확정된 키워드로 bid-searcher 에이전트를 통해 병렬 검색하라.

bid-searcher에게 다음 컨텍스트를 반드시 전달하라:
- 검색 키워드 목록 (우선순위 포함)
- 기업 프로필 요약:
  - 업종, 핵심 분야
  - 매출 규모 / 평균 수주 금액
  - 주요 인증/자격
  - 기존 실적 기관 목록
  - 보유 기술 스택

MCP Tools 활용: g2b_search_bids, kstartup_search_announcements 등

검색 결과에서:
- 마감된 공고는 자동 제외됨 (MCP 서버 + bid-searcher 이중 필터)
- 적합도 스코어링이 적용됨 (bid-searcher 5항목 정량 스코어링)

## Phase 3: 반복 심층 분석

### Step 4: 공고 선택
적합도 순으로 정렬된 결과를 다음 형식으로 제시하고, 사용자가 상세 분석할 공고를 1건 이상 선택하게 하라.

```
## 검색 결과 (총 {N}건, 마감 제외 {M}건)

| # | 적합도 | 공고명 | 기관 | 예산 | 마감 | 상태 |
|---|--------|--------|------|------|------|------|
| 1 | [높음] 82점 | AI 기반 ... | 서울시 | 5억 | D-21 | |
| 2 | [높음] 75점 | 빅데이터 ... | 국토부 | 3억 | D-14 | |
| 3 | [중간] 55점 | 클라우드 ... | 행안부 | 8억 | D-7 | |

상세 분석할 공고 번호를 선택하세요 (쉼표로 구분하여 여러 건 선택 가능)
```

### Step 5: 선택 공고 심층 분석
선택된 각 공고에 대해 순차적으로 다음을 실행하라:

1. `g2b_get_bid_detail`로 상세 조회
2. 마감일 재확인
   - 이미 마감된 공고인 경우: "이 공고는 {마감일}에 마감되었습니다. 그래도 분석을 진행하시겠습니까?"
   - `deadlineStatus`가 `urgent`인 경우: "마감까지 {N}일 남았습니다. 준비 기간이 부족할 수 있습니다."
3. `download_attachment`로 첨부파일 다운로드
4. `list_downloads`로 다운로드된 파일 확인
5. Read Tool로 첨부파일 내용 읽기
   - PDF: pages 파라미터로 20페이지 단위 분할 읽기
   - 텍스트/CSV: 직접 읽기
   - HWP: 시도 후 실패 시 PDF 변환 안내
6. rfp-evaluator 에이전트로 적합성 평가
   - 공고 상세 정보 + 첨부파일 내용을 함께 전달
7. 평가 보고서 출력

### Step 6: 추가 평가 확인
평가 완료 현황을 다음 형식으로 보여주고 추가 평가 여부를 확인하라.

```
## 평가 완료 현황

| 공고 | 성공 가능성 | 핵심 전략 |
|------|-----------|-----------|
| AI 기반 ... | 72/100 | 유사 실적 강조 + AI 차별화 |
| 클라우드 ... | 58/100 | 인증 보완 필요. 컨소시엄 검토 |

아직 평가하지 않은 공고가 {N}건 있습니다.
추가로 평가하시겠습니까? (Y/N)
```

- **Yes** → Step 4로 복귀 (이미 평가한 공고는 `[평가완료]` 표시)
- **No** → Phase 4로 진행

**자동 Phase 4 진행 조건** (아래 중 하나라도 충족 시):
1. 사용자가 "No"를 선택
2. 모든 `[높음]` 등급 공고가 평가 완료됨
3. 사용자가 "종합 보고서를 작성해주세요"라고 직접 요청

## Phase 4: 종합 보고

### Step 7: 전체 분석 결과 종합
평가된 모든 공고의 결과를 종합하여 최종 보고서를 작성하라.

보고서에 포함할 내용:
- 기업 프로필 요약
- 평가된 모든 공고의 비교표

```
## 공고별 비교표

| 항목 | 공고 A | 공고 B | ... |
|------|--------|--------|-----|
| 공고명 | | | |
| 기관 | | | |
| 예산 | | | |
| 마감 | | | |
| 성공 가능성 | /100 | /100 | |
| 핵심 강점 | | | |
| 주요 리스크 | | | |
```

- 최종 추천 순위 (평가 점수 기반)
- 공고별 제안 방향 요약
- 종합 준비 체크리스트
