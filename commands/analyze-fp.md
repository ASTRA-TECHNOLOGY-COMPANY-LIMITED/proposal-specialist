# 기능점수(FP) 산정 및 대가 산출

$ARGUMENTS 에 지정된 문서를 분석하여 기능점수를 산정하고 SW 개발 대가를 산출하라.

## 인자 파싱

$ARGUMENTS 형식: `<문서경로> [옵션...]`

- **문서경로**: RFP 파일, 요구사항 문서, 또는 generate-toc으로 생성한 목차.md 경로
  - 경로가 디렉토리인 경우 Glob Tool로 하위에서 RFP(PDF), 요구사항(MD/XLSX) 파일을 자동 식별
- **옵션** (선택):
  - `간이법` 또는 `정규법`: 산정 방법 지정 (기본값: 간이법)
  - `이윤율=N`: 이윤율 지정 (기본값: 15%)
  - `직접경비=N`: 직접경비 금액 지정 (기본값: 0원)

인자가 없으면:
1. `data/output/` 디렉토리에서 Glob Tool로 최근 생성된 목차.md를 찾는다
2. `data/business/` 디렉토리에서 RFP 문서를 찾는다
3. 파일을 찾지 못하면 사용자에게 경로를 요청한다: "FP 산정 대상 문서 경로를 입력해주세요. (RFP, 요구사항 문서, 또는 목차.md)"

## 실행 절차

### Step 1: 문서 읽기

Read Tool로 대상 문서를 읽는다.

**목차.md인 경우** (generate-toc 출력물):
- YAML 프론트매터에서 사업명, 발주처, 사업예산, 평가방식을 추출
- 각 절(섹션)의 기능 요구사항을 식별하여 FP 산정에 활용
- 목차.md의 rfp_path에서 원본 RFP도 참조하여 상세 요구사항을 보강

**RFP(PDF)인 경우**:
- pages 파라미터로 분할 읽기 (20페이지 단위)
- 핵심 추출 항목:
  - 사업명, 발주처, 사업예산, 사업기간
  - 기능 요구사항 목록 (SFR)
  - 성능 요구사항 (응답시간, 동시접속자 등)
  - 인터페이스/연계 요구사항 (연계 기관 목록)
  - 보안 요구사항
  - 운영환경 요구사항 (OS, 브라우저, 모바일 등)

**요구사항 문서(MD/텍스트)인 경우**:
- 기능 목록, 화면 목록, ERD 등에서 기능 요구사항 추출

### Step 2: 기업 시드 확인 (선택)

`data/seed/` 디렉토리에서 Glob Tool로 기업 시드 문서를 찾는다.
시드가 있으면 Read Tool로 읽어 기업의 기존 자산(재사용 가능 기능)을 파악한다.
시드가 없어도 FP 산정은 진행 가능하다.

### Step 3: 산정 방법 확인

사용자에게 산정 방법을 확인한다:

```
FP 산정을 시작합니다.

- **사업명**: {사업명}
- **발주처**: {발주기관}
- **식별된 기능 요구사항**: 약 {n}개 항목

산정 옵션:
1. **간이법** (기본) — 평균 가중치 적용, 빠른 산정
2. **정규법** — DET/RET/FTR 기반 정밀 산정

이윤율: {기본 15%} / 직접경비: {기본 0원}

이대로 진행할까요? (옵션 변경이 필요하면 말씀해주세요)
```

사용자 확인 후 다음 단계로 진행한다.
옵션에서 이미 지정한 경우 확인 없이 바로 진행한다.

### Step 4: fp-analyzer 에이전트 호출

fp-analyzer 에이전트에게 다음 컨텍스트를 전달하여 FP 산정을 위임한다.

**사업 컨텍스트:**
- 사업명: {사업명}
- 발주처: {발주기관}
- 사업예산: {예산} (RFP에 명시된 경우)
- 사업기간: {기간}

**요구사항 컨텍스트:**
- 기능 요구사항 목록: {SFR 전체}
- 성능 요구사항: {성능 관련 요구}
- 인터페이스 요구사항: {연계 기관 및 방식}
- 보안 요구사항: {보안 관련 요구}
- 운영환경 요구사항: {OS, 브라우저, 모바일 등}

**산정 옵션:**
- 방법: {간이법/정규법}
- 이윤율: {N}%
- 직접경비: {금액}원

**기업 자산 (시드가 있는 경우):**
- 재사용 가능 기능: {기존 솔루션/플랫폼에서 재사용 가능한 기능 목록}

### Step 5: 결과 저장

fp-analyzer의 결과를 다음 경로에 Write Tool로 저장한다:

`data/output/{사업명_경로}/fp-analysis.md`

- `{사업명_경로}`는 사업명에서 공백과 특수문자를 제거한 파일시스템 안전 문자열
- 기존 목차.md와 같은 디렉토리에 저장 (목차.md가 있는 경우)

### Step 6: 결과 보고

저장 완료 후 다음 형식으로 요약 보고한다:

```
## FP 산정 완료

- **사업명**: {사업명}
- **산정 방법**: {간이법/정규법}
- **저장 위치**: `data/output/{사업명_경로}/fp-analysis.md`

### 기능점수 요약

| 기능유형 | 기능수 | FP |
|---------|-------|-----|
| ILF | {n}개 | {fp} |
| EIF | {n}개 | {fp} |
| EI | {n}개 | {fp} |
| EO | {n}개 | {fp} |
| EQ | {n}개 | {fp} |
| **합계** | **{총}개** | **{총FP} FP** |

### 대가산정 요약

| 항목 | 금액 |
|------|-----|
| 미조정 개발원가 | {금액}원 |
| 보정계수 곱 | {곱} |
| 보정 후 개발원가 | {금액}원 |
| 직접경비 | {금액}원 |
| SW 개발비 | {금액}원 |
| 이윤 ({N}%) | {금액}원 |
| 개발 대가 | {금액}원 |
| **최종 대가 (VAT 포함)** | **{금액}원** |

### 합리성 검증

{Sanity Check 결과 요약 — OK 또는 주의 항목 표시}

### 특기사항

{산정 시 가정한 사항이나 주의점}
```

### Step 7: 다음 단계 안내

```
### 다음 단계

상세 내역은 저장된 보고서를 확인해주세요:
`data/output/{사업명_경로}/fp-analysis.md`

목차가 아직 없다면 제안서 목차를 생성하세요:
`/proposal-specialist:generate-toc {RFP경로} {시드경로}`

이미 목차가 있다면 제안서 작성을 진행하세요:
`/proposal-specialist:write-section data/output/{사업명_경로}/목차.md`
```
