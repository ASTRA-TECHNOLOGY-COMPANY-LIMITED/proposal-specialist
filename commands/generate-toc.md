# 제안서 목차 생성

$ARGUMENTS 에 지정된 RFP 문서와 기업 시드 문서를 분석하여 제안서 목차를 생성하라.

## 인자 파싱 및 문서 확보

$ARGUMENTS 형식: `<RFP경로> [시드경로]`

### RFP 문서 확보

다음 순서로 RFP 문서를 확보한다:

1. **$ARGUMENTS에서 확인**: 인자가 1개 이상이면 첫 번째를 RFP 경로로 사용한다
2. **자동 탐색**: 인자가 없으면 다음을 순차 탐색한다
   - `data/business/` 디렉토리에서 Glob Tool로 RFP/제안요청서 파일(`.pdf`, `.md`) 검색
   - 파일이 여러 개 발견되면 목록을 보여주고 사용자에게 선택을 요청한다
3. **사용자에게 요청**: RFP를 찾지 못하면 대화형으로 요청한다:
   "RFP(제안요청서) 파일 경로를 입력해주세요. (예: data/business/사업명/제안요청서.pdf)"
   - 사용자가 경로를 입력하면 해당 파일로 계속 진행한다

- 경로가 디렉토리인 경우 Glob Tool로 하위 파일 목록을 조회하여 RFP(PDF)와 시드(MD) 파일을 자동 식별한다

### 기업 시드 문서 확보

다음 순서로 시드 문서를 확보한다:

1. **$ARGUMENTS에서 확인**: 인자가 2개이면 두 번째를 시드 경로로 사용한다
2. **자동 탐색**: 시드 인자가 없으면 `data/seed/` 디렉토리에서 Glob Tool로 `.md` 파일을 찾는다
   - 1개 발견되면 자동 사용한다
   - 여러 개 발견되면 목록을 보여주고 사용자에게 선택을 요청한다
3. **사용자에게 요청**: 시드를 찾지 못하면 대화형으로 요청한다:
   "기업 시드 문서 경로를 입력해주세요. (예: data/seed/회사명/시드.md)
   기업 소개서, 기술문서 등 기업 역량을 파악할 수 있는 파일이면 됩니다."
   - 사용자가 경로를 입력하면 해당 파일로 계속 진행한다

## 실행 절차

### Step 1: RFP 읽기

Read Tool로 RFP를 읽는다.

**PDF 파일인 경우** 다음 순서로 선택적으로 읽는다:
1. `pages: "1-5"` — 표지, 목차 (사업명 추출)
2. `pages: "6-10"` — 사업개요 (사업기간, 예산, 추진체계)
3. 제안요청 내용 섹션 — 사업범위, 요구사항 총괄표, 요구사항 목록
4. 상세 요구사항 — 주요 기능 요구사항(SFR) 상세 내용
5. 제안서 작성요령 — RFP가 명시한 제안서 목차 및 작성방법 표
6. 제안안내 — 평가방법, 평가기준 배점표

100페이지 초과 PDF는 목차, 요구사항, 평가기준 섹션만 선택적으로 읽는다.

**마크다운/텍스트 파일인 경우** 직접 읽는다.

**디렉토리인 경우** 공고서 PDF와 제안요청서 PDF를 모두 읽는다.

RFP에서 반드시 추출할 항목:
- 사업명, 공고번호
- 사업기간, 사업예산, 추정가격
- 입찰방식 (제한경쟁, 일반경쟁 등), 계약방법 (협상, 적격심사 등)
- 수요기관 (발주처)
- RFP가 명시한 제안서 목차/작성 구조 (있는 경우)
- 기술능력 평가항목 및 배점 (표 전체)
- 요구사항 분류별 수량 및 핵심 요구사항 번호
- 참가 자격 요건

### Step 2: 시드 문서 읽기

Read Tool로 기업 시드 마크다운 파일을 읽는다.

시드에서 반드시 추출할 항목:
- 기업명, 설립연도, 기업 유형, 소재지
- 재무 현황 (매출, 자본금, 성장률)
- 보유 인증 및 자격
- 핵심 인력 (이름, 자격, 경력연수, 관련 프로젝트)
- 주요 수행 실적 (발주처, 사업명, 금액, 기간)
- 자체 플랫폼/솔루션 (이름, 기능, 기술스택)
- 핵심 강점 어필 포인트
- 기술 스택 상세

시드 디렉토리에 여러 파일이 있으면 모두 읽어 통합한다.

### Step 3: toc-generator 에이전트 호출

toc-generator 에이전트에게 다음 컨텍스트를 전달하여 목차 생성을 위임한다.

**RFP 컨텍스트:**
- 사업명: {추출한 사업명}
- 공고번호: {공고번호}
- 사업기간: {기간}
- 사업예산: {예산}
- 입찰방식: {방식}
- 계약방법: {방법}
- 발주처: {수요기관}
- RFP 명시 제안서 목차: {작성방법 표 전체 또는 "명시 없음"}
- 기술능력 평가항목 및 배점: {평가표 전체}
- 요구사항 총괄: {분류별 수량표}
- 주요 요구사항: {SFR 목록 및 핵심 요구사항 상세 내용}
- 참가 자격 요건: {자격 요건}

**기업 컨텍스트:**
- 기업명: {기업명}
- 기업 유형: {유형}
- 설립연도: {연도}
- 재무 현황: {매출, 성장률}
- 보유 인증: {인증 목록}
- 핵심 인력: {인력 테이블 요약}
- 주요 실적: {실적 테이블}
- 자체 솔루션: {플랫폼/솔루션 설명}
- 핵심 강점: {강점 어필 포인트}
- 기술 스택: {기술 목록}

### Step 4: 출력 파일 저장

toc-generator의 결과를 다음 경로에 Write Tool로 저장한다:

`data/output/{사업명_경로}/목차.md`

- `{사업명_경로}`는 toc-generator가 프론트매터에 포함한 파일시스템 안전 문자열이다
- 디렉토리가 없으면 Write Tool이 파일을 쓰면서 자동 생성된다
