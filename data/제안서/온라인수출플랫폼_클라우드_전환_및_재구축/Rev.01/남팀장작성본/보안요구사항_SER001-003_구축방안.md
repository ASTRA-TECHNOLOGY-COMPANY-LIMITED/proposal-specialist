# 4. 기밀 보장

---

## 4.1 보안 관리 체계

착수부터 사업 종료 시까지 현장 보안 점검 및 교육을 진행하며, 고객사의 보안 규정을 수용하고 준수합니다.

### 보안 관리 체계

보안 관리 체계는 **관리적 보안**, **기술적 보안**, **물리적 보안**의 3개 축으로 구성하며, 점검결과에 따른 취약점 보완 및 재검사를 중심에 두고 지속적으로 운영합니다.

#### 관리적 보안

- 프로젝트 투입 인력에 대한 지속적인 보안 교육 실시
- 인력, 문서 등 접근 권한 관리
- 보안 서약서 작성

#### 기술적 보안

- 프로젝트 참여 인력의 PC 보안 관리
- 어플리케이션 보안 관리
- 고객사 보안 규정/지침 반영
- ID 및 패스워드 관리 강화
- 취약점 점검 지원

#### 물리적 보안

- 프로젝트 수행 사무실의 출입 통제
- 기타 필요한 물리적 조치를 통한 보안 관리

### 인적 자원 보안

| 인적 자원 보안 | 인적 자원 보안 프로세스 수립 | 기대 효과 |
|----------|----------|---------|
| 직무기술서와 위탁 계약의 조항 및 조건에 보안 관련 사항 정의 | 주기적 보안 관리 교육 | 보안 체제의 효과성에 대한 감사 |
| 정보처리설비에 대한 보안 역할과 책임에 대한 협정에 서명 | 효과적인 보안 정책 수립/적용 | 주기적인 검증을 통한 보안 유지 |
| 계약 종료 또는 변경 사항 발생시 모든 장비의 회수와 접근권한 제거를 보장하기 위한 책임을 이행하는 준거 확립 | 사고 발생 방지책 강구 | 사전 예방을 통한 보안 위협의 최소화 |

### 준수 대상 보안 규정 및 지침

발주기관 및 상위 기관의 보안 규정과 지침을 준수하며, 국가정보원의 보안성검토 결과에 따른 보안정책 및 조치사항을 적용합니다.

- 국가정보원의 국가 정보보안 기본지침
- 중소벤처기업부, 중소벤처기업진흥공단의 보안 규정 및 지침
- 개인정보보호법 등 개인정보보호 관련 법규, 지침
- 행정안전부장관이 정하는 「소프트웨어 개발보안 가이드」
- 행정기관 및 공공기관 정보시스템 구축·운영 지침
- 중소벤처기업진흥공단의 「정보화 용역사업 보안관리 실무가이드」
- 소프트웨어 보안약점 진단가이드
- 「공공기관의 개인정보보호에 관한 법률」에 의한 개인정보 보호지침 등

## 4.2 관리적 보안

고객사 보안 정책을 준수하며, 참여 인원, 내부 자료 및 산출물에 대한 지속적인 보안 사고 방지 활동을 실시합니다.

> ※ 관리 방안은 개인정보보호법, 개인정보의 기술적·관리적 보호조치 기준, 정보통신망법 등 관련 법규를 준수합니다.

### 관리적 보안 관리 방안

| 구분 | 관리 방안 | 비고 |
|------|--------|------|
| **정책 및 지침** | 인적·물적 자원에 보안 정책 및 지침 수립 후 적용, 보안 정책에 따른 수시 보안 진단 실시 | |
| **인적 자원 관리** | 프로젝트 인력 투입 시 각 개인의 친필 서명이 들어간 보안서약서 제출, 인원 교체 발생 시 정보보안책임자에게 보고(임의 인원 교체 금지), 사업 시작 전 또는 인원 교체 시 보안 교육 후 참여 | |
| **비밀 보안 준수** | 사업 수행 중 취득한 지식은 비밀 보안 준수, 사업 완료 후에도 취득한 지식에 대한 비밀 보안 준수 | |
| **취득 정보 유출 금지** | 수행 과정 중 취득한 자료와 정보는 발주사의 승인 없이 외부 유출 또는 누설 금지, 법적 책임이 있는 대표자용 보안확약서 및 참여자용 보안 확약서 작성 및 제출 | |
| **내부 자료 관리** | 사업 수행에 필요한 내부 자료는 복사 및 외부 반출 금지 | 단, 발주사 승인 시 프로그램소스 등 산출물 보관을 위한 보조기억매체 제외 |
| **용역 산출물 보안** | 사업 수행 시 생산되는 모든 산출물은 지정된 산출물 저장소 또는 PC에 저장, 사업 종료 시 고객사에서 지정하는 정보보안 담당자의 입회 하에 완전 폐기 또는 반납, 일체의 성과물은 제3자 대여/양도/판매 금지 | 지정된 자료 관리 PC는 외부와의 Network 차단 |
| **개인정보보호** | 개인정보 보호 및 관리 사항은 개인정보보호법 및 개인정보의 기술적/관리적 보호조치 기준을 준수하여 관리 실시, 개인식별 정보는 암호화하여 DB에 저장하며 소스코드에 직접 코딩하지 않음, 개인정보처리 방침에 따라 개인정보 처리/파기하며 백업데이터 생성은 금지 | |
| **저작권** | 저작권을 침해할 수 있는 소프트웨어 및 문서의 보유 및 사용 금지 | |

### 개인정보보호 관리 방안

수집되는 개인정보는 「개인정보보호법」 및 「개인정보의 안전성 확보조치 기준」에 따라 관리되도록 개발하며, 다음 사항을 이행합니다.

- 개인정보 수집은 최소화하고, 고유식별번호 등은 암호화하여 저장
  - 중진공 표준 솔루션 **DGuard**를 활용한 DB 암호화 적용
- 개인정보 접속기록을 저장하고, 비정상적인 접속여부 등을 관리
- 수집된 개인정보의 파기 절차를 구현
- 개인정보의 안정성을 고려하여 특정 정보는 가명정보, 합성정보 등으로 변환 수행
  - 개발·테스트 환경에서는 실제 개인정보 대신 합성정보를 생성하여 활용

### 보안관리계획 제출

사업 수행과 연관된 문서, 인원, 장비 등에 대한 물리적·관리적·기술적 보안대책 및 '누출금지 대상정보'에 대한 보안관리 계획을 사업수행계획서에 포함하여 제출합니다.

- 발주자의 보안정책(정보보안지침, 개인정보보호지침 등)을 준수하며, 이를 위반하였을 경우 「외주 용역사업 보안 특약」에 따라 관련자에 대하여 조치하고 손해배상 책임을 짐
- 해당 정보 누출시 「국가를 당사자로 하는 계약에 관한 법률 시행령」 제76조에 따라 부정당업자로 등록

## 4.3 물리적/기술적/망접근 보안

내/외부망 접근 관리, 시설에 대한 출입 관리, 장비 반출 관리 등 지속적인 물리적 보안 사고 방지 활동을 실시합니다.

### 물리적/기술적 보안 관리 방안

| 구분 | | 관리 방안 | 비고 |
|------|------|--------|------|
| **물리적 보안** | 장비 반입/반출 관리 | 외부 반입 장비 악성 코드 감염 확인, 사용자 PC 백신 프로그램 최신 상태 유지, 정기적인 바이러스 및 해킹도구 검사 실시, 노트북·USB 등 장비 관리 절차 준수 | |
| | 시설 보안 | 프로젝트 수행 사무실 주요 장비 설치 장소에 대한 출입 보안 실시 | |
| | 사무 환경 관리 | 중요 문서 보관을 위한 잠금 장치가 되어 있는 보관함 사용, 문서의 보안 등급을 부여하고 권한 관리 | |
| | 자료 폐기 | 사업 종료 후 산출물은 대외 보안의 정도에 따라 삭제 또는 폐기, 노트북·보조기억매체 등 전자적으로 기록된 자료는 「정보시스템 저장매체 불용처리 지침」 준수, 완벽한 자료의 삭제를 위해 데이터삭제 S/W 사용 또는 Format 후 반출 | |
| **기술적 보안** | 시스템 보안 | 시스템 Password의 주기적인 변경 관리 실시 | |
| | 바이러스 보안 | 사용자 PC 백신 프로그램 최신 상태 유지 | |
| | 데이터 보안 | 불의 사고 대비 시스템·파일 피해 최소화를 위해 백업 정책 수립 및 실행 | |
| **내외부망 접근 보안** | | 제안사 시스템은 방화벽 또는 원격접근통제시스템을 활용함, 필요시 발주사 정보보안 S/W 설치, 악성코드 전염·해킹 등이 발생하지 않도록 규정 준수하며 미준수에 대한 책임에 동의, AWS 기반 개발환경을 구성하기 위해 필요사항은 협의하여 예외처리 필요함 | |

## 4.4 인력 보안

사업 참여 인력에 대한 주기적인 보안 교육 및 점검 활동을 통해 보안 사고 예방 활동을 실시합니다.

### 인력 보안 관리 방안

| 구분 | 관리 방안 | 비고 |
|------|--------|------|
| **공통** | 사업 시작 전 또는 인원 교체 시, 보안 교육 후 참여, 보안 의식 제고를 위한 주기적인 교육 실시를 통해 보안 사고 예방(월 1회), 보안 정책 위반 시 '사업자 보안 위규 처리 기준'에 따라 행정 조치 | |
| **사업 착수 시** | 프로젝트 인력 투입 시 각 개인의 친필 서명이 들어간 보안서약서 제출 | |
| **사업 수행 시** | 노트북·PC·USB·외장HDD 등의 개인 저장 장치의 반입/반출 통제(발주사 정책에 따름), 개발 참여 인원 철수 시 특별 보안 교육 실시, 개발 참여 인원 철수 시 개인 PC 반출 절차 적용 및 접속 계정 삭제, 인원 교체 발생 시 정보보안 책임자에게 보고(임의 인원 교체 금지), 보안인식 강화를 위해 주기적 보안 교육 실시, 발주사 요구 보안교육 참석 | |
| **사업 종료 시** | 사업 참여자 활용한 PC는 프로젝트 자료 사용이 불가능하도록 조치, 사업 관련 자료 보유 금지하며, 위반 시 법적 책임이 포함된 "대표자용 보안확약서" 및 "참여자용 보안확약서" 제출 | 자료 사용 불가능 조치 방법은 고객사와 협의 후 확정 |
