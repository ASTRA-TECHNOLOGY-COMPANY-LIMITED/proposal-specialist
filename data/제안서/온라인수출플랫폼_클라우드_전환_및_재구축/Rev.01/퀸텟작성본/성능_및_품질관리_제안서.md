# Ⅳ. 성능 및 품질

## 1. 성능 요구사항

### 1.1 어플리케이션 성능

#### 1.1.1 응답속도 보장 방안(자료 디스플레이 시간)

체감성능을 보장하기 위한 Architecture와 개발표준을 설계하고, 클라이언트 환경과 서버환경 각 구간별 응답시간을 점검/튜닝하여 최적의 속도를 내기 위한 방안을 수립합니다.

**▶ 성능 목표**

| 구분 | 목표값 | 비고 |
|------|--------|------|
| 웹페이지 응답시간 | 3초 이내 | 사용자 요청 시점부터 최초 결과 표시 |
| 일일 배치 업무 | 10분 이내 | 정기 배치 처리 기준 |
| 월 배치 업무 | 1시간 이내 | 대량 데이터 배치 기준 |
| 시스템 자원 사용율 | 최대 90% 미초과 | CPU, 메모리, 디스크 기준 |

**▶ 기술영역 별 구축 방안**

**① Cloud Architecture**

- Lambda 및 ECS Scale-out 리소스 분리를 통한 성능확보
- NCP 서비스 및 서버리스 아키텍처를 활용한 비용 절감
- 온라인 트랜잭션은 API Gateway 기반 Lambda Function으로 처리하고, 배치/인터페이스는 ECS Container 기반 Auto Scaling으로 구성하여 워크로드 특성에 맞는 최적 아키텍처를 적용합니다.

**② Application 최적화**

네트워크 사용량을 줄이고 웹페이지 실행 속도를 높이기 위한 기술요소를 도입하며, 이미지 분할, URL 캐싱 등 솔루션을 적용합니다.

| 최적화 영역 | 주요 방안 | 상세 내용 |
|-------------|-----------|-----------|
| Redirect 방지 | 불필요한 Redirect 제거 | 302, 301 등의 불필요한 Redirect를 제거하여 요청 처리 단계를 최소화 |
| Caching | Cache 설정 및 캐싱 적용 | Cache 만료주기 등 태그 설정, URL 캐싱, 이미지 분할(솔루션 적용) |
| N/W Cost 축소 | 네트워크 비용 절감 | HTTP/2 적용, CDN(Content Delivery Network), 도메인 샤딩, 압축전송 적용, Webpack 등 도구를 이용한 컨텐츠 분할 |
| Rendering 최적화 | 빠른 렌더링 처리 | Virtual DOM을 이용한 빠른 렌더링 처리, CSR(Client Side Rendering)의 페이지별 최적안 설계 |

- 처리 형태가 느린 경우 사용자에게 사전 알림 방안을 수립합니다(세부내역은 중진공과 협의).

**③ QA(품질검증)**

- **체감성능 점검**: 단말, 브라우저 중심의 서비스 속도 측정
  - Mobile 단말 속도점검(LTE, 5G, WIFI)
  - QA 인력을 통한 매뉴얼 점검
- **성능 점검**: NCP CloudWatch를 통한 응답속도 지연구간 분석
  - 성능/가용성 점검
  - NCP CloudWatch 기반 모니터링을 통한 Application 튜닝 최적화

#### 1.1.2 사용자 증가에 대한 성능 보장(동시 접속자수)

워크로드의 확장성 구성/설정을 분석하여 테스트 대상/범위를 정의하고 리소스 별 확장 Configuration과 Parameter에 대한 확장성 테스트(Scale In/Out)를 진행합니다.

**▶ 동시 처리 목표**

| 구분 | 목표값 |
|------|--------|
| 초당 처리 건수(TPS) | 50건 이상 |
| 동시 접속자 | 1,000명 이상 정상 유지 |
| 임계치 알림 | 최대 동시접속자 수 90% 이상 시 서비스 지연 안내 페이지 제공 |

**▶ 성능 검증 절차**

| 과업(Task) | 활동(Activity) | 설명 | 산출물 |
|------------|---------------|------|--------|
| 성능 검증 준비 | 대상 업무 및 시나리오 선정, 동시 접속자 산정 및 리소스 사양/목표치 선정 | 테스트 시나리오 선정 및 목표 임계치 설정, 동시 접속자 수 예상 리소스 사양 및 사용률 목표 산정 | 테스트 계획서 |
| 성능 검증 | 목표 TPS 부하주입/응답시간 측정, Cloud 리소스별 성능 지표/모니터링 측정 | 테스트 수행 및 성능 지표 모니터링(측정), Jmeter(부하툴), CloudWatch(모니터링) 사용 | 테스트 결과 수집 |
| 분석 및 최적화 | 리소스 조정, Application 병목/지연 조치 | 성능 기준을 고려하여 측정된 성능 지표 Data 분석, 리소스 부족/과할당 등을 식별하여 성능 기준에 맞는 최적화 수행 | 튜닝 내역서 |

**▶ 분석 및 최적화 프로세스**

1. Jmeter를 통한 부하주입 및 목표 RPS/TPS, 응답시간, Batch/Lambda 처리시간 측정
2. NCP 리소스(ALB/NLB, ECS/Fargate, RDS(MySQL)/Redis, EBS/S3) 환경에서 시나리오별 성능지표 모니터링/측정(CloudWatch)
3. 서비스 관점 성능기준 준수 여부(응답/처리시간) 확인
4. NCP 리소스 성능지표 분석(성능 기준 대비 여유 및 병목/부족 여부)
5. 병목/지연 원인 식별: Application Logic, DB Query(Slow 등), I/F Latency 등 외부요인
6. 리소스 수량조정/Spec Up 또는 Spec Down을 통한 최적화
7. 성능테스트 수행 및 분석 과정 반복

> **참고**: Application 측면의 병목/지연 Point는 성능 테스트와 병행하여 PJT팀에서 식별/분석/조치를 수행합니다.

#### 1.1.3 시스템 안정성(서비스 연속성 보장)

피크 타임 및 과부하에 따른 시스템 안정성을 점검하고, NCP 환경을 고려한 테스트를 수행하여 서비스의 연속성 방안을 수립 및 실행합니다.

**▶ 서비스 연속성(확장성) 방안**

| 과업(Task) | 활동(Activity) | 산출물 |
|------------|---------------|--------|
| 확장성 정책 | 성능 확인 화면 선별 등의 테스트 대상 정의, 테스트 시나리오 및 시나리오 별 임계 부하량 선정 등 테스트 범위 정의 | 성능 테스트 계획서 |
| 확장성 구성 | Cloud 리소스별 확장구성, 확장성 테스트/정상 동작 여부 확인 | 성능 테스트 결과 보고서 |
| 확장성 설정 | Cloud 리소스 설정변경, Cloud 리소스 최적화 | 성능 테스트 결과 보고서 |

**▶ 확장성 검증 체계**

- **PJT별 환경**: PJT 요건을 반영한 별도 환경(Jmeter)에서 임계(확장) 부하를 성능 테스트 동일 대상 및 시나리오로 수행
- **NCP 리소스**: 확장성 정책, 확장성 구성, 확장성 설정을 NCP 리소스에 적용
- **모니터링 및 검증**: 임계 부하 인지, 확장성 정책/구성/설정의 정상 동작 여부(Scale-out), 서비스 정상 여부, 원복 정상 동작 여부 등 확인

**▶ 확장성 테스트 절차**

1. Scalability 설정/검증을 위한 임계 부하 테스트
2. 워크로드 SA 구성/설정 분석
3. 확장성 테스트 대상 정의
4. 리소스 별 확장 Conf/Parameter 체크
5. 임계 부하 주입(성능 테스트 동일 방식)
6. 임계 여부 확인(Monitoring/Alerting)
7. 확장 여부 검증(Scale-out)
8. 확장 리소스 원복 여부 확인
9. 부하 해제

## 2. 품질 요구사항

### 2.1 품질관리

#### 2.1.1 품질관리 개요

전문 품질 인력과 제안사의 조직적 품질 역량을 통해 체계적인 품질관리를 수행합니다.

> **"산출물 신뢰성 확보, 고객 만족 달성, 품질 경쟁력 제고"**

**▶ 품질관리 3대 특징**

| 구분 | 특징 | 상세 내용 |
|------|------|-----------|
| 특징 01 | 최적화된 개발방법론 적용 | 다양한 프로젝트에서 검증된 현장 중심의 실용적 개발 방법론 적용, 핵심 Task 중심의 절차 적용 |
| 특징 02 | 품질보증 활동 강화 | 단계 말 집중 검토에서 선제적 산출물 수시 검토 수행, 요구사항 Baseline 설정과 지속적인 변경관리를 통한 기능 구현 정확성 확보 |
| 특징 03 | 전사 품질지원 체계 활용 | 전사 품질/방법론/테스트/튜닝 지원, Risk 진단 및 프로세스 이행 점검 |

**▶ 품질 관리 활동**

| 활동 영역 | 주요 내용 |
|----------|-----------|
| 품질 계획 수립 | 객관적/정량적 품질 목표 수립, 품질 목표와 Align된 품질 활동 계획 수립 |
| 품질 보증 및 통제 활동 | 예방 차원의 품질 활동 강화(교육 강화), 실질적 품질 확보에 집중(PL 및 발주사(고객 현업)에 의한 컨텐츠 품질 검토), 정량적 분석 및 개선 활동 |
| 품질 조직 구성 | 아키텍처, 테스트 지원 조직 협력, 전사 품질 역량 활용 |

**▶ 단계 별 품질 예상 Risk**

| 공통 | 분석 | 설계 | 개발 및 테스트 |
|------|------|------|--------------|
| 표준 및 가이드 불명확 | 현행 시스템 분석 자료 미흡 | 지속적인 요구사항 변경 및 추가 | 설계/개발 산출물 간 불일치 |
| 표준 절차의 복잡성 | 요구사항의 미 도출 및 불명확 | 의사결정 지연으로 인한 설계 미흡 | 테스트 시나리오/케이스 미흡 |
| 과다한 산출물 작성 | 업무 범위 및 R&R에 대한 이견 발생 | 설계 산출물에 대한 실질적인 품질 검토 부족 | 테스트 데이터 불충분 |
| 산출물에 대한 고객 검토 미흡 | | | 결함에 대한 조치 여부 불명확 |

#### 2.1.2 품질보증 계획

품질 계획, 목표, 요건에 대한 지속적인 품질 관리를 시행하며, 구축 이후에는 결과 검토를 통한 피드백, 수정/보완 등을 통하여 서비스의 품질을 보증합니다.

**▶ 품질관리 절차**

품질계획수립 → 품질관리활동 → 변경관리 → 품질검토(재검토/심사통과) → 서비스 런칭 → 지속관리

**▶ 프로젝트 품질관리 내용**

| 활동 | 내용 |
|------|------|
| 품질보증 계획수립 | 품질보증 목표와 품질보증활동 계획을 수립 |
| 표준 및 절차 수립 | 각 산출물 작성 표준 및 작성절차, 사용양식 정의 |
| 형상관리 | 도입 소프트웨어 및 문서, 장비 등에 대한 형상 항목을 식별하고 변경 사항을 통제하여 상태를 기록하고 유지 |
| 테스트 | 단위, 통합, 시스템 테스트 실시 |
| 워크스루 | 프로젝트 자체로 산출물에 대한 검토를 실시하여 결함을 조기에 발견하고 조치 |
| 정기협의 | 서비스 런칭 이후 정기 협의로 능동적인 품질 보증 활동 수행 |
| INFRA 보완 | 서비스 런칭 이후 시스템에 대해 INFRA 보완 |

**▶ 전사 품질관리 활동**

| 활동 | 내용 |
|------|------|
| 사내감리 | 각 단계를 종료할 때 전문 감리인이 문서 검토 및 인터뷰 실시를 통해 산출물과 프로젝트 진행 시 위험 요소를 도출 |
| 제품검사 | 설치 시점에 시스템 기능 및 성능을 점검하여 보완케 하고, 구현 시점에 최종 검사를 실시 |
| 고객 만족도 조사 | 프로젝트 구축 또는 서비스 런칭 후 고객 만족도 조사를 실시하여 고객불만사항을 파악하고 개선 대책을 통하여 문제점을 해결하고 고객 감동을 추구 |

#### 2.1.3 품질관리 절차

서비스의 일정(진척도) 관리를 통해 프로젝트 담당자 및 관리자에게 현재 진척 및 진행률에 대한 정보를 수집 및 전달합니다.

**▶ 진척 관리 원칙**

- 진도 관리의 기준은 각 업무 별로 확인된 사항을 Bottom-up 방식으로 취합하여 PM/PL이 관리합니다.
- 진도 관리 결과 일정에 영향을 미칠 경우가 발생했을 때는 주요 Issue로 관리됩니다.
- 진도에 수정이 발생하였을 경우에는 즉각적으로 반영하고, 해당 업무팀에 즉각 통보합니다.

**▶ 진척 관리 절차**

| 구분 | 절차 | 상세 내용 |
|------|------|-----------|
| 업무 단위 관리 | Task 단위 진척도 산출 | 팀별 PL의 주도하에 진도율 확인 및 진척관리 문서 작성 |
| 서비스 전체 관리 | PM 확인 및 전체 진도율 산출 | 전체 주간 미팅에서 진도율 보고 |

**▶ 계획 및 실적 관리**

| 단계 | 활동 |
|------|------|
| 프로젝트 전체일정 수립 | 프로젝트 시작 단계 작성 |
| 진행률 등록 및 확인 | 주기적 진행률 확인 |
| 진행률 최종 확인 | 적용된 진행률 최종 확인, 지연될 때 이슈 등록 및 이슈관리 진행 |
| 실적 관리 | 계획 대비 실적관리 리포팅, 주간 진척 관리 보고서 작성 |
