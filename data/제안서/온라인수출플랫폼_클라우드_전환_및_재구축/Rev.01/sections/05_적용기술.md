### 2.3 적용기술

#### 2.3.1 시스템 아키텍처

##### (1) 하이브리드 클라우드 아키텍처

![전체 시스템 아키텍처(하이브리드 클라우드)](../images/05_system_architecture_hybrid_cloud.png)

본 시스템은 **민간클라우드와 중진공 IDC를 IPsec VPN으로 연결하는 하이브리드 클라우드** 아키텍처로 구성합니다. 공개 SW 중심으로 시스템 SW(OS, WEB, WAS) 환경을 구성하고, 시스템 SW의 버전 업그레이드, 소프트웨어 변경에도 서비스의 안정성이 보장되도록 구현합니다(SFR-001).

**시스템 구성 총괄표**

| 계층 | 구성요소 | 민간클라우드 (신규) | 중진공 IDC (기존) | 비고 |
|------|--------|-------------|-------------|------|
| **WEB** | Nginx | 2식(국문/영문) + 1식(관리자) | 1식(통합정보시스템) | 리버스 프록시, 로드밸런싱, 정적 파일 서빙 |
| **Front-end** | Nexacro N V24 | 1식(관리자) | 1식(통합정보) | 행정/관리자 사이트 전용(RFP 지정) |
| **WAS** | Tomcat | 2식(국문) + 2식(영문) + 1식(관리자) | 1식(통합정보) | 전자정부 표준프레임워크 5.0 기반 |
| **DB** | MariaDB / Oracle | MariaDB 2식 HA 구성 | Oracle(기존 활용) | 신규 MariaDB, 기존 Oracle 병행 |
| **NAS** | 스토리지 | 2식(운영/개발) | - | 파일 스토리지 |
| **OS** | Linux | 전 서버 | 기존 활용 | - |
| **JDK** | Open JDK | 17/21/23 | - | 발주사 협의 가능 |
| **개발언어** | Java, JS 등 | - | - | - |

##### (2) 개발 및 운영 환경 분리

애플리케이션의 개발을 위한 클러스터와 운영을 위한 클러스터의 구성을 분리합니다(SFR-001).

| 환경 | 네트워크 | 용도 | 접근 정책 |
|------|---------|------|---------|
| **개발 환경** | 고비즈코리아 내부망 전용 | 소스변경 사항 배포, 기능 테스트 | 개발자 IP 제한 접근 |
| **운영 환경 (내부)** | 고비즈코리아 내부망 | 관리자/통합정보시스템 서비스 | 중진공 업무망 접근 |
| **운영 환경 (외부)** | 고비즈코리아 DMZ망 | 국문/영문사이트 외부 서비스 | 공개 접근(WAF 적용) |

##### (3) 네트워크 구성(VPN/DMZ)

![네트워크 구성도(VPN/DMZ)](../images/05_network_architecture_vpn_dmz.png)

외부 시스템 연계 시 IPSec VPN, 전용선 등의 보안통신을 구성합니다(SFR-001).

| 네트워크 구간 | 보안 구성 | 용도 |
|---------|---------|------|
| **Public → DMZ** | SSL/TLS, WAF, IP 필터링 | 국문/영문사이트 외부 접속 |
| **DMZ → Private Cloud** | 방화벽, 접근제어 | WEB→WAS→DB 내부 통신 |
| **Private Cloud → 중진공 IDC** | IPsec VPN 터널 | 통합정보시스템, Oracle DB 연계 |
| **중진공 업무망 → 관리자** | VPN, 업무망 내부 | 관리자사이트 접근(SFR-001) |
| **영문사이트 → CDN** | HTTPS, 엣지 캐시 | 해외 바이어 이미지 캐쉬서버(SFR-005) |

> **컨소시엄 강점**: 아스트라비전은 APEX/FECT 플랫폼을 Nginx/Tomcat 기반으로 운영 중이며, 퀸텟시스템즈는 AWS ISV Partner로서 클라우드 네트워크 보안 설계 경험을 보유하고 있습니다.

#### 2.3.2 핵심기술 상세

##### (1) 클라우드 네이티브 기술 (SFR-001)

공개 SW 중심으로 시스템 SW 환경을 구성하고, 클라우드 네이티브 운영을 위한 모니터링 및 보안관제 환경을 구축합니다.

| 기술 영역 | 적용 기술 | 구현 내용 | 컨소시엄 경험 |
|---------|---------|---------|----------|
| **컨테이너 오케스트레이션** | Kubernetes | 클러스터 자동 확장, 서비스 중단 최소화를 위한 오토스케일링 정책 마련(SFR-001) | APEX/FECT K8s 실운영 |
| **모니터링** | Prometheus/Grafana | 클라우드 네이티브 운영을 위한 모니터링 및 보안관제 환경 구축 | APEX 250개 기업 모니터링 |
| **보안 정책** | 방화벽/접근제어 | 현 운영환경을 준용하여 방화벽 정책, 서버 접근권한 등 보안정책, 백업 및 모니터링 정책 적용 | FTA Korea 8년 보안 운영 |

##### (2) 컨테이너 기반 배포 (SFR-002)

![컨테이너 배포(Blue/Green, CI/CD)](../images/05_container_deployment_bluegreen.png)

컨테이너화된 애플리케이션을 쉽고 안전하게 배포하기 위해 자동화 기반의 컨테이너 배포를 구현합니다(SFR-002).

| 기능 | 구현 내용 | 대응 요구사항 |
|------|---------|----------|
| **이미지 리파지토리** | 컨테이너별 이미지 지정 및 리소스 할당, 로드밸런싱을 통해 서비스 공개 | SFR-002 |
| **컨테이너 레지스트리** | HTTPS를 통해 암호화되어 전송, 네임스페이스를 사용하여 레지스트리 정의 | SFR-002 |
| **무중단 배포** | Blue/Green배포, Rolling배포, Canary배포 등 적합한 무중단 배포 방식 제시 | SFR-002 |
| **CI/CD 파이프라인** | 지속적 통합과 배포를 지원하는 파이프라인 기능 제공, 자동/수동/스케줄러 트리거 지원 | SFR-001 |
| **배포 이력관리** | 배포에 대한 이력관리 및 롤백 지원 | SFR-001 |

> **컨소시엄 강점**: 아스트라비전은 APEX/FECT 플랫폼에서 Docker 컨테이너 + GitHub Actions CI/CD 파이프라인을 실운영하고 있으며, Blue/Green 배포 전략을 통한 무중단 배포를 수행하고 있습니다.

##### (3) API Gateway (SFR-003)

![API Gateway 구조도](../images/05_api_gateway_architecture.png)

API 호출을 위한 토큰 발급 및 인증, 엔드포인트별 API 호출 인증 및 인가, 접근 정책 등을 관리합니다(SFR-003).

| 기능 | 구현 내용 | 대응 요구사항 |
|------|---------|----------|
| **인증/인가** | API 호출을 위한 토큰 발급 및 인증, 접근 정책(특정 클라이언트의 API 호출 불허 등) 처리 | SFR-003 |
| **API 라우팅** | 동일 API를 클라이언트나 마이크로서비스에 따라 다른 엔드포인트를 통해 서비스, 여러 클라이언트 마이크로서비스별로 엔드포인트 제공 | SFR-003 |
| **공통 로직 처리** | 로깅, 인증 등 공통 기능을 중복 개발 또는 처리하지 않도록 요청과 응답의 표준화 및 공통 로직 처리 | SFR-003 |
| **프로토콜 변환** | 동일 API를 HTTP, REST, XML 웹 서비스 등 클라이언트와 마이크로서비스별로 상이한 프로토콜로 서비스하기 위한 프로토콜 변환 처리 | SFR-003 |
| **모니터링** | API 호출 패턴 분석, API 호출 실행 및 접근 상태 분석, 요청 IP/클라이언트 일시 등 API 호출에 대한 로깅 및 모니터링 | SFR-003 |

##### (4) CDN 서비스 (SFR-005)

영문사이트의 상품이미지 캐쉬서버를 별도 구현하여 클라우드 CDN서비스를 적용합니다(SFR-005). 미국, 캐나다 등 거리가 먼 곳에서도 원활하게 고비즈코리아에 접속 가능합니다.

| CDN 적용 범위 | 구현 내용 |
|---------|---------|
| **이미지 캐시** | 상품 이미지, 배너 이미지 등 정적 자원 엣지 캐싱 |
| **HTTPS 암호화** | SSL/TLS 인증서 적용으로 보안 통신 보장 |
| **글로벌 엣지** | 미국/캐나다/유럽/아시아 주요 리전 PoP 활용 |
