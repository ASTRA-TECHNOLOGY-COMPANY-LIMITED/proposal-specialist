## VI. 프로젝트지원

### 6.1 품질보증방안

본 컨소시엄은 품질보증 전담 조직을 구성하고, 분석/설계/구현/테스트/이행 각 단계별로 체계적인 품질보증 활동을 수행합니다. 중진공, KOTRA, KTNET, KISTI, 질병관리청 등 다수 공공기관 검수를 통과한 경험을 바탕으로, 본 사업에서도 발주처의 요구 품질 수준을 충족합니다.

> **[요구사항 대응]** QUR-001(품질관리 일반사항), QUR-002(기능 구현 정확성), QUR-003(상호운영성), QUR-004(신뢰성, 확장성)

---

#### 6.1.1 품질보증 조직

품질관리 조직과 운영절차를 다음과 같이 구체적으로 편성합니다.

| 역할 | 담당자 | 주요 업무 |
|------|--------|---------|
| **품질관리 총괄** | PMO 남의석 | 품질관리 계획 수립, 품질 지표 관리, 품질 점검 총괄 |
| **품질 검증(기술)** | CTO 이상진 | 코드 리뷰, 아키텍처 적합성 검증, 성능 기준 충족 확인 |
| **품질 검증(기능)** | PL 오대산 | 기능 요구사항 충족 여부 검증, AIMS 기준 품질 심사 |
| **품질 검증(UI/UX)** | PL 김지예 | 화면 디자인 일관성, 사용성 검증, UI/UX 가이드 준수 |
| **품질 검증(도메인)** | PL 민세준 | 수출/무역 관련 기능 정확성, 업무 로직 검증 |

> **PL 오대산의 AIMS 심사원 역량**: ISO/IEC 42001:2023 AIMS AI 경영시스템 심사원 자격을 보유한 오대산 이사가 AI 기능을 포함한 전체 시스템의 품질 검증을 수행합니다.

---

#### 6.1.2 단계별 품질보증 활동

QUR-001에 따라, 품질활동의 제반절차 및 산출물을 명시한 품질관리계획을 제안서 및 사업수행계획서에 상세하게 기술하고, 이에 근거하여 체계적이고 효과적인 프로젝트를 진행합니다.

**가. 분석 단계 품질활동**

| 품질 활동 | 점검 항목 | 산출물 |
|---------|---------|--------|
| 요구사항 검토 | 요구사항 완전성, 일관성, 추적성 | 요구사항 검토 결과서 |
| 현행 시스템 분석 검증 | 현행 기능/데이터/인프라 분석 정확성 | 현행분석 검토 결과서 |
| 요구사항 추적 매트릭스 | RFP 요구사항 100% 매핑 확인 | 요구사항 추적표(RTM) |

**나. 설계 단계 품질활동**

| 품질 활동 | 점검 항목 | 산출물 |
|---------|---------|--------|
| 아키텍처 리뷰 | 클라우드 전환 아키텍처 적합성, 확장성 | 아키텍처 리뷰 결과서 |
| DB 설계 리뷰 | 데이터 모델 정규화, 성능, 무결성 | DB 설계 리뷰 결과서 |
| 화면 설계 리뷰 | UI/UX 가이드라인 준수, 사용성 | 화면 설계 리뷰 결과서 |
| 인터페이스 설계 리뷰 | 연계 시스템 인터페이스 적합성 | 인터페이스 리뷰 결과서 |

**다. 구현 단계 품질활동**

| 품질 활동 | 점검 항목 | 산출물 |
|---------|---------|--------|
| 코드 리뷰 | 코딩 표준 준수, 시큐어코딩 적용 | 코드 리뷰 결과서 |
| 단위 테스트 | 모듈별 기능 정상 동작 확인 | 단위테스트 결과서 |
| 정적 분석 | 소스코드 보안 취약점, 코드 품질 | 정적 분석 보고서 |
| 기능 구현 정확성 검증 | QUR-002에 따른 요구사항 제공 여부 확인 | 기능 검증 결과서 |

> QUR-002에 따라, 시스템은 제공되기로 한 요구사항을 모두 제공하며, 기능 구현 정확성은 사용자가 직접 테스트를 수행함으로써 평가합니다. 신규 기능 개발로 인해 기존 기능 및 성능에 영향을 미치지 않도록 회귀 테스트를 반드시 실시합니다.

**라. 테스트 단계 품질활동**

| 품질 활동 | 점검 항목 | 산출물 |
|---------|---------|--------|
| 통합 테스트 | 시스템 간 연계, 데이터 정합성 | 통합테스트 결과서 |
| 성능 테스트 | 응답시간, 동시접속자, TPS | 성능테스트 결과서 |
| 보안 테스트 | 보안 취약점 점검, 시큐어코딩 검증 | 보안점검 결과서 |
| 상호운영성 검증 | QUR-003에 따른 기술표준 호환 | 상호운영성 검증서 |
| 인수 테스트(UAT) | 실제 사용자 기반 기능 검증 | UAT 결과서 |

> QUR-003에 따라, 목표시스템은 본 사업과 관련된 정보시스템 및 기술표준과의 상호 운영성을 확보하며, 행정기관 및 공공기관 정보시스템 구축 운영 지침에 따른 표준 기술을 적용합니다.

**마. 이행 단계 품질활동**

| 품질 활동 | 점검 항목 | 산출물 |
|---------|---------|--------|
| 데이터 마이그레이션 검증 | 데이터 정합성, 누락 여부 | 마이그레이션 검증서 |
| 시범운영 모니터링 | 결함 발생률, 장애 현황 | 시범운영 결과서 |
| 최종 품질 검수 | 전체 요구사항 충족 여부 확인 | 최종 품질점검 보고서 |

#### 6.1.3 품질 지표 및 측정

QUR-004에 따라, 시스템의 목표 달성을 검증할 신뢰성 있는 측정 및 평가 방법을 다음과 같이 제시합니다.

| 품질 지표 | 측정 기준 | 목표 값 |
|---------|---------|--------|
| 요구사항 충족률 | 구현된 기능 수 / 전체 요구 기능 수 x 100 | 100% |
| 결함 밀도 | 결함 수 / KLOC(천 라인) | 5건 이하/KLOC |
| 시험운영 결함율 | 결함 발생률 5% 미만 시 정상 오픈 | 5% 미만 |
| 결함지속시간 | 시험운영 기간 최대 한계값 | 1시간 이내 |
| 코드 커버리지 | 테스트 커버리지 비율 | 80% 이상 |
| 보안 취약점 | 보안점검 결과 미해결 취약점 | 0건 |

> QUR-004에 따라, 시험운영 기간동안 발견된 결함 수와 결함의 지속시간을 측정하며, 결함 발생율이 5% 이상인 경우에는 시스템 오픈기간을 연장합니다. 시험운영 기간 결함지속시간의 최대 한계값은 1시간 이내로 관리합니다.

**외부 요인 대응(QUR-001)**

본 과업범위 외의 요인(운영서버 등 정보자원)으로 인해 사업 결과에 영향이 있거나 예상되는 경우, 원인과 해결방안을 발주기관에 제시하여 위험요소를 최소화하거나 문제가 해결될 수 있도록 적극 협조합니다. DB구축을 포함하는 경우 사업완료(검수) 시 데이터품질진단 및 오류데이터 개선을 실시합니다.
