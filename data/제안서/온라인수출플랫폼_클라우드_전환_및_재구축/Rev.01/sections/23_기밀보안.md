## VI. 프로젝트지원

### 6.4 기밀보안

본 컨소시엄은 사업 수행 전 과정에서 기밀보안 체계를 수립하고, 저작권 존중, 시스템 보안성 확보, 개인정보보호 대책을 포괄하는 종합 보안 방안을 적용합니다. 특히 한국-베트남 원격 협업 환경에서의 보안 관리 경험(ODA 사업 수행)을 기반으로 원격개발 보안 방안을 구체적으로 제시합니다.

#### 6.4.1 기밀보안 체계 및 대책

**기밀보안 관리 원칙:**

본 사업에서 취급하는 모든 정보는 「국가정보보안기본지침」, 「전자정부법」, 「개인정보보호법」 등 관련 법령에 따라 관리합니다.

| 보안 영역 | 관리 항목 | 구체적 대책 |
|---------|---------|----------|
| **관리적 보안** | 보안 조직, 보안 교육, 보안 서약 | 전 참여인력 보안서약서 징구, 분기별 보안 교육 실시, 보안 담당자 지정 |
| **물리적 보안** | 작업장소, 장비, 출입통제 | 개발 장소 CCTV 설치, 외부 저장매체 반입 통제, 출입대장 관리 |
| **기술적 보안** | 네트워크, 시스템, 데이터 | VPN 기반 접근, 다단계 인증(MFA), 암호화(전송/저장), 접근 로그 관리 |
| **인적 보안** | 인력 관리, 퇴직자 관리 | 보안 등급별 접근 권한 차등 부여, 퇴직/이직 시 접근 권한 즉시 회수 |

**참여인력 보안 관리 (SER-005):**

| 항목 | 내용 |
|------|------|
| 보안서약서 | 사업 착수 전 전 인력 보안서약서 징구 (한국/베트남 동일 적용) |
| 보안 교육 | 사업 착수 시 + 분기별 정기 교육 (공공기관 보안 정책, 개인정보보호) |
| 접근 권한 | 역할 기반 접근 통제(RBAC), 최소 권한 원칙(Least Privilege) 적용 |
| 퇴직/이직 관리 | 접근 권한 즉시 회수, 보유 자료 반납, 비밀유지 의무 확인 |

**원격개발 보안 방안 (COR-002):**

한국-베트남 원격 협업 환경에서의 보안 대책을 수립합니다. 본 컨소시엄은 ODA 사업 수행 과정에서 동일한 원격 협업 환경을 운영한 경험을 보유하고 있습니다.

| 보안 항목 | 적용 방안 |
|---------|---------|
| **네트워크 보안** | IPsec VPN 전용 터널 구성, 접속 IP 화이트리스트, SSL/TLS 암호화 통신 |
| **접근 통제** | 다단계 인증(MFA) 필수, 역할 기반 접근 권한, 접속 이력 실시간 모니터링 |
| **코드 관리** | GitHub Enterprise 접근 통제, Branch Protection Rule, 코드리뷰 필수 승인 |
| **데이터 보호** | 운영 데이터 개발 환경 반입 금지, 테스트 데이터 마스킹 처리, 개인정보 비식별화 |
| **단말기 보안** | 개발 PC 보안 솔루션 설치 필수, 외부 저장매체 사용 통제, 화면 캡처 제한 |
| **감사 추적** | 전 접근 이력 로깅, 월간 보안 감사, 이상 접근 시 즉시 차단 및 보고 |

**문서 보안 (SER-006):**

| 항목 | 대책 |
|------|------|
| 문서 분류 | 기밀/대외비/일반 3등급 분류 체계 적용 |
| 문서 관리 | 형상관리 시스템(Git)을 통한 문서 버전 관리, 접근 이력 추적 |
| 인쇄/복사 통제 | 기밀 문서 인쇄 승인제, 워터마크 적용 |
| 폐기 관리 | 사업 종료 후 보유 자료 완전 삭제, 폐기 확인서 발급 |

#### 6.4.2 저작권 존중여부 명시

본 사업에서 사용하는 모든 소프트웨어는 정품 라이선스를 확보하여 사용하며, 오픈소스 소프트웨어는 라이선스 조건을 엄격히 준수합니다.

| 구분 | 관리 방안 |
|------|---------|
| **상용 소프트웨어** | 정품 라이선스 확보 후 사용, 라이선스 증빙 관리 |
| **오픈소스 소프트웨어** | 라이선스 유형(MIT, Apache, GPL 등) 사전 검토, 호환성 확인, SBOM(Software Bill of Materials) 관리 |
| **개발 산출물** | 지식재산권 공동소유(발주기관-계약상대자) 원칙 준수 (COR-003) |
| **제3자 저작물** | 이미지, 폰트, 라이브러리 등 정품 사용, 출처 명시 |

#### 6.4.3 시스템 보안성 확보 방안

본 사업에서 구축하는 시스템은 「소프트웨어 개발보안 가이드」(행정안전부)를 준수하여 설계/개발/테스트 전 단계에 보안을 내재화합니다.

**시큐어코딩 적용 (SER-004, COR-005):**

| 보안 영역 | 점검 항목 | 대응 방안 |
|---------|---------|---------|
| **입력 데이터 검증** | SQL Injection, XSS, CSRF | 입력값 검증, Prepared Statement, CSRF 토큰 |
| **인증/인가** | 세션 하이재킹, 권한 상승 | 세션 타임아웃, RBAC, 토큰 기반 인증 |
| **암호화** | 평문 전송, 약한 암호화 | TLS 1.3, AES-256 암호화, 안전한 키 관리 |
| **에러 처리** | 정보 노출, 스택 트레이스 | 사용자 정의 에러 페이지, 로그 분리 관리 |
| **코드 품질** | 하드코딩, 자원 누수 | 정적 분석 도구, 코드리뷰, 시큐어코딩 체크리스트 |

**보안 취약점 점검:**

| 점검 시점 | 점검 내용 | 도구/방법 |
|---------|---------|---------|
| 개발 단계 | 소스코드 정적 분석 | SonarQube, SpotBugs |
| 테스트 단계 | 웹 취약점 동적 분석 | OWASP ZAP, Burp Suite |
| 배포 전 | 인프라 보안 점검 | 보안 체크리스트, 취약점 스캐닝 |
| 운영 단계 | 정기 보안 점검 | 월간 보안 점검, 침해사고 대응 훈련 |

#### 6.4.4 개인정보보호 대책

본 사업은 수출기업 및 해외바이어의 개인정보를 처리하므로, 「개인정보보호법」 및 「개인정보의 안전성 확보조치 기준」(개인정보보호위원회 고시)을 엄격히 준수합니다 (SER-002).

**개인정보 보호 조치:**

| 영역 | 적용 방안 |
|------|---------|
| **수집/이용** | 최소 수집 원칙, 수집 목적 명시, 동의 절차 구현 |
| **저장/관리** | 개인정보 암호화 저장(AES-256), 접근 통제, 접근 이력 관리 |
| **제공/위탁** | 제3자 제공 시 동의, 위탁 계약 시 안전조치 의무 부과 |
| **파기** | 보유기간 경과 즉시 파기, 복구 불가능한 방법 적용 |
| **권리 보장** | 열람/정정/삭제/처리정지 요구권 보장, 공공마이데이터 연계 시 본인정보 제공 동의 |

**개인정보 영향평가:**

| 항목 | 내용 |
|------|------|
| 평가 시점 | 설계 단계 완료 시 (개인정보 처리 시스템 구축 전) |
| 평가 범위 | 국문사이트 회원정보, 영문사이트 바이어정보, 통합정보시스템 수출기업정보 |
| 평가 항목 | 개인정보 수집 적정성, 안전성 확보조치, 권리 보장 절차, 파기 절차 |
| 후속 조치 | 평가 결과에 따른 보완 조치 시행, 발주처 보고 |
