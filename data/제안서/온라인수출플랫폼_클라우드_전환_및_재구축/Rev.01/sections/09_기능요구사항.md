## III. 기술 및 기능

### 3.1 기능요구사항

본 절에서는 온라인수출플랫폼 클라우드 전환 및 재구축 사업의 13개 기능요구사항(SFR-001~SFR-013)에 대한 구현 방안을 상세히 기술합니다. 퀸텟시스템즈(총괄)와 아스트라비전 컨소시엄은 buyKOREA 실무 운영 경험(KOTRA AI무역지원센터 3개소), 영문 B2B 플랫폼 구축 경험(Gobiz Vietnam, FECT), 그리고 대용량 메시지 처리 역량(RabbitMQ 일 100만건+)을 기반으로 68개 요구사항을 빠짐없이 충족합니다.

---

#### 3.1.1 클라우드 네이티브 적용 (SFR-001)

**[요구사항 개요]**

SFR-001은 공개 SW 중심의 클라우드 네이티브 환경을 구성하고, IPsec VPN을 통한 하이브리드 클라우드를 적용하며, 개발/운영 환경 분리, 오토스케일링, CI/CD 파이프라인을 구현하는 요구사항입니다.

**[구현 방안]**

**(1) 공개 SW 기반 시스템 환경 구성**

본 사업에서는 Linux OS, Nginx(WEB), Tomcat(WAS), MariaDB(신규DB) 등 공개 SW 중심으로 시스템 환경을 구성합니다. 시스템 SW의 버전 업그레이드 및 소프트웨어 변경에도 서비스 안정성이 보장되도록 컨테이너 기반 배포 환경을 적용합니다.

| 구분 | 적용 기술 | 버전 | 비고 |
|------|---------|------|------|
| OS | Linux (Rocky/Ubuntu) | 최신 LTS | 공개 SW |
| WEB | Nginx | 최신 안정 버전 | 리버스 프록시, 로드밸런싱 |
| WAS | Tomcat | 10.x | 전자정부FW 5.0 호환 |
| DB(신규) | MariaDB | 10.11+ | 고비즈코리아 메인 DB |
| DB(기존) | Oracle | 기존 활용 | 통합정보시스템(수출) |
| JDK | Open JDK | 17/21/23 | 멀티 버전 지원 |
| Framework | 전자정부 표준프레임워크 | 5.0 | 필수 적용 |

> **컨소시엄 강점**: 아스트라비전은 전자정부 표준프레임워크 기반의 FTA Korea 시스템을 2011년부터 최초 개발하고 8년간 연속 운영한 경험을 보유하고 있습니다. ODA 사업(Gobiz Vietnam)에서도 Spring Boot + Docker + Nginx 기반의 클라우드 네이티브 아키텍처를 성공적으로 적용하였습니다.

**(2) IPsec VPN 기반 보안통신 구성**

외부 시스템 연계 시 IPsec VPN, 전용선 등의 보안통신을 구성합니다. 민간 클라우드와 중진공 IDC 간 VPN 연결을 통해 하이브리드 클라우드를 적용하며, 국정원 보안성검토에 따른 보안정책을 준수합니다.

- 중진공 업무망과 고비즈코리아 내부망 간 IPsec VPN 터널 구성
- 관리자사이트(Nexacro N V24)는 업무망에서 접속하여 행정처리 편의성 강화
- 국정원 보안성검토 결과에 따른 보안정책 적용

**(3) 개발 및 운영 환경 분리**

애플리케이션의 개발을 위한 클러스터와 운영을 위한 클러스터의 구성을 분리합니다.

| 환경 | 위치 | 용도 | 접근 범위 |
|------|------|------|---------|
| 개발 환경 | 고비즈코리아 내부망 | 소스 개발/테스트 | 개발팀 전용 |
| 운영 환경(국문/영문) | 고비즈코리아 DMZ | 대외 서비스 | 인터넷 공개 |
| 운영 환경(정보시스템) | 중진공 IDC | 내부 행정처리 | VPN 접속 |

**(4) 오토스케일링 정책**

업무서비스 부하에 따른 컨테이너 리소스 용량 및 오토스케일링 정책을 마련하여, 클러스터 자동 확장 시 서비스 중단을 최소화합니다.

- CPU/메모리 사용률 기반 수평적 오토스케일링(HPA) 적용
- 이벤트(수출박람회, 바이어매칭 행사 등) 기간 트래픽 급증 대비 사전 스케일 아웃
- 클러스터 노드 오토스케일링으로 인프라 자원 탄력적 운영

**(5) CI/CD 파이프라인**

지속적 통합과 배포를 지원하는 CI/CD 파이프라인 기능을 제공합니다.

- 파이프라인 실행: 자동(트리거), 배포자 수동, 스케줄러 기반 배치 형태 전체 지원
- 배포 이력관리 및 롤백 지원
- 코드 품질 검사, 보안 취약점 스캔, 자동 테스트를 파이프라인에 통합

> **컨소시엄 강점**: 아스트라비전은 GitHub Action 기반 CI/CD 파이프라인을 FECT, APEX 플랫폼에 실 적용하고 있으며, 퀸텟시스템즈는 CALS Platz의 빌드 파이프라인을 통해 배포 시간 40% 단축을 실증하였습니다.

#### 3.1.2 컨테이너 기반 배포 (SFR-002)

**[요구사항 개요]**

SFR-002는 컨테이너화된 애플리케이션의 자동화된 배포 구현, 컨테이너 레지스트리 제공, 무중단 배포(Blue/Green, Rolling, Canary) 지원을 요구합니다.

**[구현 방안]**

**(1) 컨테이너 이미지 관리**

각 업무서비스를 독립적으로 개발, 배포, 변경할 수 있도록 Docker 컨테이너 이미지로 구성합니다.

- 이미지 리파지토리 생성, 컨테이너별 이미지 지정 및 리소스 할당
- 로드밸런싱을 통한 서비스 공개
- 애플리케이션 파일 배포 시 실행환경의 자동 생성 및 구동 지원

**(2) 컨테이너 레지스트리**

컨테이너 이미지를 안전하게 저장 관리하기 위해 프라이빗 레지스트리를 제공합니다.

- HTTPS를 통해 암호화되어 전송
- 네임스페이스를 사용하여 컨테이너 레지스트리(Registry, 이미지 저장소) 정의
- 개발 환경 이미지와 운영에 배포되는 이미지의 레지스트리 분리
- 운영환경 이미지 전송 시 신뢰할 수 있는 HTTPS 프로토콜 사용

**(3) 무중단 배포 지원**

서비스 중단 없이 새 버전을 배포할 수 있도록 다양한 무중단 배포 방식을 제시합니다.

| 배포 방식 | 적용 대상 | 특징 |
|---------|---------|------|
| Blue/Green | 국문/영문 사이트 전체 배포 | 신/구 버전 동시 운영 후 전환, 즉시 롤백 가능 |
| Rolling | 일반 업데이트 | 순차적 교체, 서비스 연속성 보장 |
| Canary | 신규 기능 검증 | 일부 트래픽만 신버전 라우팅, 점진적 확대 |

> **컨소시엄 강점**: 아스트라비전은 Kubernetes 기반 컨테이너 오케스트레이션 환경에서 APEX 플랫폼(250개 기업 동시 서비스)을 운영하고 있으며, Docker 이미지 빌드부터 배포까지 전 과정의 자동화 경험을 보유하고 있습니다.

#### 3.1.3 API Gateway (SFR-003)

**[요구사항 개요]**

SFR-003은 API 호출을 위한 토큰 발급/인증, 엔드포인트별 접근정책, API 라우팅, 프로토콜 변환, 모니터링 기능을 요구합니다.

**[구현 방안]**

**(1) 토큰 기반 인증 및 인가**

API 호출을 위한 토큰 발급 및 인증 체계를 구축합니다. 엔드포인트별 API 호출 인증 및 인가, 접근 정책(특정 클라이언트의 API 호출 불허에 의한 접근제어 기능)을 구현합니다.

- JWT(JSON Web Token) 기반 인증/인가 처리
- API Key 발급 및 관리
- Rate Limiting, Throttling 정책 적용
- IP 기반 접근 제어

**(2) API 라우팅 및 프로토콜 변환**

동일 API를 클라이언트나 마이크로서비스에 따라 다른 엔드포인트를 통해 서비스를 제공하도록 API 라우팅을 지원합니다.

- 동일 API를 여러 클라이언트/마이크로서비스별로 엔드포인트 제공
- HTTP, REST, XML 웹 서비스 등 다양한 프로토콜 변환 처리
- 동기/비동기 API 호출 패턴 변환 기능 제공

**(3) 로깅 및 모니터링**

로깅, 인증 등 공통 기능을 중복 개발 또는 처리하지 않도록 요청과 응답의 표준화 및 공통 로직을 처리합니다.

- API 호출 패턴 분석, 접근 상태 분석
- 요청 IP/클라이언트 일시 등 API 호출에 대한 로깅 및 모니터링
- 비정상 호출 탐지 및 알림

> **컨소시엄 강점**: 아스트라비전은 FECT 플랫폼에서 REST API 기반 기관 연계(관세청 UNI-PASS, 대한상공회의소)를 구현한 경험이 있으며, RabbitMQ 기반 일 100만건 이상의 비동기 메시지 처리 아키텍처를 운영하고 있습니다.

#### 3.1.4 국문사이트 개발 (SFR-004)

**[요구사항 개요]**

SFR-004는 고비즈코리아 국문사이트 재개발로서, 회원영역(공인인증서/이지패스/SSO), 수출지원, 상품서비스, UI/UX 재설계를 요구합니다.

**[구현 방안]**

![국문사이트 메인화면 와이어프레임(개념)](../images/01_korean_site_wireframe.png)

**(1) 회원영역**

- 회원가입, 로그인, 마이페이지 조회 기능 구현
- 유형별 회원분류 처리(개인회원, 기업회원)
- 공인인증서 로그인, 중소벤처24 로그인 기능(이지패스) 개발
- 중소벤처24와 통합SSO/IM 연동처리 (SSO SW는 발주사에서 제공)

**(2) 수출지원**

국내 중소기업 수출 지원 사업의 온라인 신청/접수 기능을 구현합니다.

| 수출지원 사업 | 기능 |
|------------|------|
| 공동물류 | 기업 신청, 접수 |
| 온라인수출플랫폼 | 기업 신청, 접수 |
| 전자상거래 수출지원 | 기업 신청, 접수 |

**(3) 상품서비스**

- 상품등록, 상품조회, 상품검색, 상품카테고리 기능 구현

**(4) 정보제공 및 부가서비스**

- 수출초보기업 대상 특화상담신청, 특화상담센터 운영
- 해외바이어 매칭서비스 연계
- 고비즈코리아 안내, FAQ, Q&A, 사이트맵 등

**(5) UI/UX 재설계 및 재적용**

- UI/UX 공통가이드('디지털 정부서비스 UI/UX 가이드라인', '24.2.29 배포) 준수
- 사용자 중심 UI/UX 개편으로 중소기업 온라인 수출 강화
- 반응형 웹 디자인 적용(PC, 모바일, 태블릿)

> **컨소시엄 강점**: 아스트라비전은 KOTRA AI무역지원센터 3개소(인천, 강원, 경북)를 운영하며 buyKOREA 제품등록, 기업 온보딩 실무 프로세스를 숙지하고 있습니다. 국문사이트의 수출지원 사업 신청/접수 흐름에 대한 현장 경험을 바탕으로, 사용자 편의성이 극대화된 UI/UX를 설계합니다.

#### 3.1.5 영문사이트 개발 (SFR-005)

**[요구사항 개요]**

SFR-005는 고비즈코리아 영문사이트(buyKOREA) 재개발로서, SNS 간편 회원가입(구글/링크드인/페이스북), 상품조회, 쇼핑몰 연동(buyKorea/tradeKorea/카페24/아마존), 토스페이먼츠 결제, CDN을 요구합니다.

**[구현 방안]**

![영문사이트 메인화면 와이어프레임(개념)](../images/02_english_site_wireframe.png)

**(1) 회원영역**

- 회원가입, 로그인, 마이페이지 조회 기능 구현
- 구글, 링크드인, 페이스북 등 SNS 간편 회원가입 (지원하는 SNS간편가입은 발주사와 협의)
- OAuth 2.0 기반 소셜 로그인 연동

**(2) 상품/서비스**

- 상품조회, 상품카테고리, 기업정보확인, 구매문의, 상품주문, 상품주문이력 기능 구현
- 중소기업 매칭서비스 제공

**(3) 공공 및 민간 쇼핑몰 링크 기능 개발**

해외바이어가 한국 중소기업 제품에 쉽게 접근할 수 있도록 다양한 쇼핑몰과 연동합니다.

| 연동 대상 | 연동 방식 | 비고 |
|---------|---------|------|
| buyKorea | 물품연동 (각 사이트 API 활용) | 공공 쇼핑몰 |
| tradeKorea.com | 물품연동 (각 사이트 API 활용) | 공공 쇼핑몰 |

| 아마존 | 아마존 API 활용 양방향 상품 연동 | 민간 쇼핑몰 |

- 상품 상세 보기 기능 등에서 민간 쇼핑몰 링크 기능 제공
- 민간 쇼핑몰 링크 장애일 경우, 대체 페이지 자동 표시

**(4) 토스페이먼츠 결제 연동**

- 상품별 토스페이먼츠 결제 연동 (토스페이먼츠 API 활용)
- 판매자(중소기업)의 가입링크 등 활용안내페이지 제공
- 고비즈코리아를 통한 매출액 등 통계 표시
- 고비즈코리아와 토스페이먼츠사 고객키 관리 (운영/정산 기능은 토스페이먼츠사에서 제공)

**(5) CDN 적용**

이미지 캐쉬서버를 별도 구현하여, 클라우드 CDN 서비스를 적용합니다.

- 전세계에서 접속지연 없이 접속할 수 있도록 CDN(Content Delivery Network) 서비스 적용
- 영문사이트의 상품이미지 캐쉬서버 구축
- 미국, 캐나다 등 거리가 먼 곳에서도 원활한 접속 보장

**(6) UI/UX 재설계**

- UI/UX 공통가이드('디지털 정부서비스 UI/UX 가이드라인') 준수
- 해외바이어가 원하는 정보 및 상품을 손쉽게 확인할 수 있도록 사이트 재개발

> **컨소시엄 강점**: 아스트라비전은 Gobiz Vietnam(베트남 공공 온라인 수출플랫폼) 구축 경험과 FECT(영문 관세무역 포털) 운영 경험을 통해 해외 바이어 대상 B2B 영문 사이트의 UX 설계 노하우를 축적하였습니다. APEX 플랫폼의 바이어-셀러 매칭 시스템(Precision@10: 0.80~0.90)과 B2B 챗봇(4개국어 지원)을 통해 해외 바이어 전환율을 극대화합니다.

#### 3.1.7 공공마이데이터 연계 (SFR-008)

**[요구사항 개요]**

SFR-008은 행정안전부 공공마이데이터 유통시스템과의 연계를 통해 본인기업정보를 안전하게 전송받고, PDS(보안저장소) 기반 데이터 관리 체계를 구축하는 요구사항입니다.

**[구현 방안]**

![공공마이데이터 연계 흐름도](../images/03_public_mydata_flow.png)

**(1) 본인기업정보 요청 API 세트 구현**

- 행정안전부 공공마이데이터 유통시스템간에 이용기관 발급신청 API를 활용하여 데이터를 안전하고 신뢰할 수 있는 방식으로 전송 받을 수 있도록 표준 규격을 준수하여 연계

**(2) 본인기업정보 수신 API 세트 구현**

- 제공요구권 수취 API, 데이터 이용내역 API 등을 활용하여 중진공 내 이를 설치/구현하고 본인기업정보를 전송받을 수 있도록 연계

**(3) 묶음정보 등록 및 API 연계 테스트**

- 데이터 요청문을 생성하여 제공시스템에 전송하고, 정상적으로 묶음정보가 발급되는지를 테스트 수행

**(4) 공공 마이데이터 업무 메뉴 및 화면 신설**

| 메뉴 | 기능 |
|------|------|
| 마이데이터 전송 | 전송요구 등록, 철회, 변경 (본인 인증 및 제공 동의 절차) |
| 마이데이터 관리 | 전송요구 철회, 변경 및 제공, 공공 마이데이터 내역 조회 |
| 마이데이터 이용 내역 | PDS를 이용하여 전송 제공 요구서 및 요청 결과 내용 조회 |

**(5) PDS 수신 정보 활용**

- PDS로 수신한 행정정보를 행정 및 사업관리서비스에서 확인 가능
- 수신한 행정정보는 구분 정보에 따라 제공하도록 구현

**(6) 보안저장소(PDS) 개발**

- 행정안전부 가이드라인에 따라 보안저장소(PDS) 개발
- 암호화된 데이터 저장 및 접근 통제

> **컨소시엄 강점**: 아스트라비전은 APEX 플랫폼에서 본인기업정보 기반 수출전략 수립 기능을 구현하였으며, API 연계(REST API, JPA) 기반 외부 시스템 통합 경험이 풍부합니다.

SFR-012는 통합정보시스템(수출) 재개발을, SFR-013은 통합정보시스템(수출)과 행정(내부, 관리자)사이트 간의 연계처리를 요구합니다.

**(2) 권한관리**

- 기존 통합정보시스템과 동일한 기능 개발

**(3) 수출마케팅 실적 조회 관리**

**(5) 중복수혜방지관리**

- 부정당업자 등록 및 관리

- 기존 데이터 마이그레이션 지원

**(7) 통합정보시스템(수출) 연계 (SFR-013)**

**(8) API 기반 연계처리**

- 통합정보시스템(수출)과 행정(내부, 관리자)사이트 연계
- API를 통해 데이터 연계처리
- 데이터 통신간 암호화 처리

**(9) 장애 및 로그관리**

- 장애 발생에 따른 로그 관리
- 재처리 기능 개발
- 자동 알림 및 모니터링
